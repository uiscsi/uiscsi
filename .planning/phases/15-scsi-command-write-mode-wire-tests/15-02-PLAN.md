**TestSCSICommand_ImmediateDataMatrix** -- table-driven with 4 subtests:

1. **"ImmediateData=Yes"** (SCSI-01, FFP #16.1.1):
   - NegotiationConfig: ImmediateData=true, InitialR2T=false, MaxRecvDSL=512
   - Write 512 bytes (EDTL=512, fits in one MaxRecvDSL segment)
   - Target handler: since ImmediateData=Yes and InitialR2T=No, the command carries immediate data. No R2T needed if EDTL fits in one segment. Target reads no Data-Out (data is in SCSI Command), sends SCSI Response success.
   - Assert SCSI Command PDU: Write=true, DataSegmentLen=512, ExpectedDataTransferLength=512, Final=true (all data in immediate)

2. **"ImmediateData=No"** (SCSI-02, FFP #16.1.2):
   - NegotiationConfig: ImmediateData=false, InitialR2T=true
   - Write 512 bytes
   - Target handler: Send R2T for full EDTL, read Data-Out until F-bit, send response.
   - Assert SCSI Command PDU: Write=true, DataSegmentLen=0 (no immediate data), ExpectedDataTransferLength=512, Final=true (F-bit set because no unsolicited data follows when InitialR2T=Yes)

3. **"ImmediateData=No/InitialR2T=Yes/no-unsolicited"** (SCSI-04, FFP #16.2.2):
   - NegotiationConfig: ImmediateData=false, InitialR2T=true
   - Write 1024 bytes
   - Target handler: Send R2T for full EDTL, consume Data-Out, send response.
   - Assert SCSI Command PDU: DataSegmentLen=0 (no immediate data). Also assert NO Data-Out PDUs with TTT=0xFFFFFFFF (no unsolicited data at all). This is the "no unsolicited data" verification.

4. **"ImmediateData=No/InitialR2T=No/no-immediate"** (SCSI-05, FFP #16.2.3):
   - NegotiationConfig: ImmediateData=false, InitialR2T=false, FirstBurstLength=1024
   - Write 512 bytes
   - Target handler: Read unsolicited Data-Out until F-bit (since InitialR2T=No, unsolicited data is allowed but not immediate), send response.
   - Assert SCSI Command PDU: DataSegmentLen=0 (no immediate data even though unsolicited Data-Out is allowed). Assert Data-Out PDUs with TTT=0xFFFFFFFF exist (unsolicited data was sent as Data-Out, not in SCSI Command). This distinguishes immediate data (in SCSI Command) from unsolicited Data-Out.

Each subtest uses newWriteTestSetup with appropriate NegotiationConfig, dialWithOverrides with matching overrides, sess.WriteBlocks for the write, then rec.Sent(pdu.OpSCSICommand) to capture and assert.

Use t.Run for each subtest. Each test function starts with a conformance comment referencing the requirement ID and FFP number.

**TestSCSICommand_UnsolicitedFBit** -- table-driven with 2 subtests for SCSI-03 and SCSI-07:

1. **"EDTL=DataSegmentLen/F-bit-unsolicited"** (SCSI-03, FFP #16.2.1):
   - NegotiationConfig: ImmediateData=true, InitialR2T=false, MaxRecvDSL=512, FirstBurstLength=512
   - Write exactly 512 bytes (EDTL = MaxRecvDSL = FirstBurstLength = 512)
   - Target handler: All data fits in SCSI Command immediate data (EDTL=FBL=MaxRecvDSL). No Data-Out expected. Send SCSI Response.
   - Assert SCSI Command PDU: DataSegmentLen=512, EDTL=512, Final=true. Assert NO unsolicited Data-Out PDUs sent (TTT=0xFFFFFFFF). The F-bit in SCSI Command means "all unsolicited data is in this PDU" which is true since EDTL=DSL.

2. **"InitialR2T=Yes/F-bit-command"** (SCSI-07, FFP #16.3.1):
   - NegotiationConfig: ImmediateData=true, InitialR2T=true, MaxRecvDSL=512
   - Write 1024 bytes
   - Target handler: SCSI Command has immediate data (ImmediateData=Yes) but InitialR2T=Yes means no unsolicited Data-Out follows. Target sends R2T for remaining data (EDTL - immediate), reads Data-Out, sends response.
   - Assert SCSI Command PDU: Final=true (no unsolicited Data-Out follows), DataSegmentLen=512 (immediate data up to MaxRecvDSL), EDTL=1024, Write=true.

**TestSCSICommand_FirstBurstLength** -- table-driven with the 5 boundary scenarios from D-04, all for SCSI-06 (FFP #16.2.4):

All scenarios use ImmediateData=Yes, InitialR2T=No (unsolicited data allowed). MaxRecvDSL=512 throughout.

1. **"EDTL<FirstBurstLength"** (D-04 scenario 1):
   - FirstBurstLength=2048, EDTL=512 (write 512 bytes)
   - All data fits in immediate (512 <= MaxRecvDSL). No Data-Out needed.
   - Assert: SCSI Command DataSegmentLen=512, Final=true. No Data-Out PDUs at all.

2. **"EDTL=FirstBurstLength"** (D-04 scenario 2):
   - FirstBurstLength=1024, MaxRecvDSL=512, EDTL=1024 (write 1024 bytes)
   - Immediate data = 512 (MaxRecvDSL), then 1 unsolicited Data-Out of 512 with F-bit. Total unsolicited = 1024 = FBL. No R2T needed.
   - Target handler: Read unsolicited Data-Out until F-bit. Send response.
   - Assert: SCSI Command DataSegmentLen=512. Total unsolicited (immediate+Data-Out) = 1024. Last unsolicited Data-Out has Final=true.

3. **"EDTL>FirstBurstLength"** (D-04 scenario 3):
   - FirstBurstLength=1024, MaxRecvDSL=512, EDTL=2048 (write 2048 bytes)
   - Immediate = 512, unsolicited Data-Out = 512 (total unsolicited = 1024 = FBL), then R2T for remaining 1024.
   - Target handler: Read unsolicited Data-Out until F-bit, send R2T for remaining, read solicited Data-Out, send response.
   - Assert: SCSI Command DataSegmentLen=512. Total unsolicited <= FirstBurstLength (1024). Solicited Data-Out PDUs present.

4. **"EDTL=MaxRecvDSL"** (D-04 scenario 4):
   - FirstBurstLength=2048, MaxRecvDSL=512, EDTL=512 (write 512 bytes)
   - Single PDU: immediate data = 512. Done.
   - Assert: SCSI Command DataSegmentLen=512, Final=true. No Data-Out PDUs.

5. **"EDTL=2xFirstBurstLength"** (D-04 scenario 5):
   - FirstBurstLength=512, MaxRecvDSL=512, EDTL=1024 (write 1024 bytes)
   - Immediate = 512 (= FBL = MaxRecvDSL). No more unsolicited (FBL exhausted). Target sends R2T for remaining 512.
   - Target handler: Send R2T for 512 at offset 512, read Data-Out, send response.
   - Assert: SCSI Command DataSegmentLen=512, Final=true (FBL=MaxRecvDSL so immediate exhausts FBL, F-bit set). Solicited Data-Out present with TTT != 0xFFFFFFFF.

For each scenario:
- Use newWriteTestSetup with the NegotiationConfig
- dialWithOverrides with ImmediateData=Yes, InitialR2T=No
- Write the specified number of bytes via sess.WriteBlocks
- Capture SCSI Command PDU via rec.Sent(pdu.OpSCSICommand)
- Assert DataSegmentLen, EDTL, Final, Write fields
- Where applicable, assert unsolicited vs solicited Data-Out counts using TTT=0xFFFFFFFF filter

Use the sendR2TAndConsume and sendSCSIResponse helpers from helpers_test.go in the HandleSCSIFunc where appropriate.
