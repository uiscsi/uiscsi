<objective>
Fix test suite timeout by making the mock target behave like a real target. The root cause: every test session's Close() attempts a graceful Logout PDU exchange, but the test pipe goes silent after the test body — nobody reads the Logout request, nobody sends LogoutResp. The 5s context timeout burns on every test (44 tests x 5s = 220s).

The fix: test cleanup spawns a goroutine that reads from the target pipe and auto-responds to the Logout PDU with a proper LogoutResp — exactly like a real iSCSI target would. The session does a real graceful shutdown, completing instantly instead of timing out. Zero production code changes.

Purpose: The session test suite must pass with `go test -timeout 120s` (default) including all Phase 4 write path tests.

    **1. Add `respondToLogout` helper to session_test.go** (after the existing helpers, before the first Test function):

    ```go
    // respondToLogout reads PDUs from the target conn and auto-responds to a
    // Logout request with a successful LogoutResp. This lets Close() complete
    // a real graceful logout instead of waiting 5s for a response that never
    // comes. Any non-Logout PDUs (e.g., NOP-Out keepalives) are silently
    // consumed. Returns when the conn is closed or after responding to Logout.
    func respondToLogout(conn net.Conn) {
        for {
            raw, err := transport.ReadRawPDU(conn, false, false)
            if err != nil {
                return // conn closed or broken — nothing to do
            }
            opcode := pdu.OpCode(raw.BHS[0] & 0x3f)
            if opcode != pdu.OpLogoutReq {
                continue // consume keepalive pings, etc.
            }
            // Extract ITT from BHS bytes 16-19 and send LogoutResp.
            itt := binary.BigEndian.Uint32(raw.BHS[16:20])
            resp := &pdu.LogoutResp{
                Header: pdu.Header{
                    OpCode_:          pdu.OpLogoutResp,
                    Final:            true,
                    InitiatorTaskTag: itt,
                },
                Response: 0, // success
            }
            bhs, err := resp.MarshalBHS()
            if err != nil {
                return
            }
            _ = transport.WriteRawPDU(conn, &transport.RawPDU{BHS: bhs})
            return
        }
    }
    ```

    **Why this works:** When a test completes, `t.Cleanup` runs. The goroutine starts reading from targetConn. Then `sess.Close()` checks `wasLoggedIn && s.err == nil`, enters `logout()`, sends LogoutReq via the write pump. The goroutine reads it from targetConn, sends back LogoutResp. The session's read pump delivers it to `logout()` via the router. `logout()` returns immediately. Total time: microseconds.

    **Edge cases handled:**
    - If the test body already closed the session or broke the connection, `respondToLogout` gets a read error and returns silently.
    - If keepalive NOP-Out PDUs are in the pipe, they're consumed before the Logout.
    - If `s.err != nil` (connection broke during test), `Close()` skips logout entirely and `respondToLogout` hits a read error.
