    ```go
    // PDUHandler processes a received PDU and returns response PDUs.
    type PDUHandler func(conn *TargetConn, req *transport.RawPDU, decoded pdu.PDU) error

    // TargetConn wraps a connection to a single initiator, providing
    // helper methods for sending response PDUs.
    type TargetConn struct {
        nc     net.Conn
        mu     sync.Mutex // serialize writes
        statSN uint32
    }

    // SendPDU marshals and sends a PDU to the initiator.
    func (tc *TargetConn) SendPDU(p pdu.PDU) error

    // SendRaw sends a pre-built RawPDU.
    func (tc *TargetConn) SendRaw(raw *transport.RawPDU) error

    // NextStatSN returns and increments the StatSN counter.
    func (tc *TargetConn) NextStatSN() uint32

    // MockTarget is an in-process iSCSI target for testing.
    type MockTarget struct {
        listener net.Listener
        handlers map[uint8]PDUHandler // keyed by opcode byte
        mu       sync.Mutex
        conns    []*TargetConn
        done     chan struct{}
        wg       sync.WaitGroup
    }
    ```

    Constructor:
    ```go
    // NewMockTarget starts a mock target listening on 127.0.0.1:0.
    // It automatically accepts connections and dispatches PDUs to handlers.
    func NewMockTarget() (*MockTarget, error) {
        ln, err := net.Listen("tcp", "127.0.0.1:0")
        // start accept loop goroutine
        // for each conn: start serve loop goroutine
        // serve loop: ReadRawPDU -> DecodeBHS -> dispatch to handler
    }
    ```

    Methods:
    - `Addr() string` -- returns listener address for Dial
    - `Handle(opcode uint8, h PDUHandler)` -- registers handler for opcode
    - `Close() error` -- closes listener, waits for goroutines
    - `Conns() []*TargetConn` -- returns active connections (for assertions)

    Built-in convenience handlers (methods on MockTarget):
    - `HandleLogin()` -- registers OpLoginReq handler that:
      1. Parses login request, reads CSG/NSG/transit
      2. Responds with LoginResp(StatusClass=0, Transit=true)
      3. Handles security phase (AuthMethod=None) and operational negotiation
      4. Accepts all proposed operational parameters (echoes back target values using login.DecodeTextKV/EncodeTextKV)
      5. Sets TSIH=1 in final response
    - `HandleSCSIRead(lun uint64, data []byte)` -- responds to SCSI Command with Read flag: sends DataIn PDU(s) with provided data + SCSIResponse(status=0)
    - `HandleSCSIWrite(lun uint64)` -- responds to SCSI Command with Write flag: sends R2T if needed, collects DataOut, sends SCSIResponse(status=0)
    - `HandleLogout()` -- responds to LogoutReq with LogoutResp(response=0)
    - `HandleNOPOut()` -- responds to NOP-Out with NOP-In (keepalive)

    **Concurrent testing with synctest:** For tests involving timing-sensitive mock target behavior (e.g., delayed responses, timeout verification, keepalive exchanges), use `testing/synctest.Run()` to wrap the test function. This enables deterministic time control via `synctest.Wait()` instead of `time.Sleep()`. The MockTarget's goroutines (accept loop, serve loop) will participate in the synctest bubble when started inside `synctest.Run()`. Apply this pattern in target_test.go and advise conformance tests to use it where timeout behavior is tested.

    Helper functions:
    - `buildRawPDU(p pdu.PDU) (*transport.RawPDU, error)` -- marshals PDU to RawPDU (extracted from session_test.go)
    - `BuildLoginResp(req *pdu.LoginReq, keys []login.KeyValue, transit bool, csg, nsg uint8) *pdu.LoginResp` -- exported for custom login test scenarios
