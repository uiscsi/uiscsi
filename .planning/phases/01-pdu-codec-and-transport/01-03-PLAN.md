    2. Create `internal/transport/framer.go` with PDU framing functions:
       - Define `RawPDU` struct:
         ```go
         type RawPDU struct {
             BHS          [48]byte
             AHS          []byte   // nil if no AHS
             DataSegment  []byte   // copied out, caller-owned (D-05)
             HeaderDigest uint32   // 0 if not present
             DataDigest   uint32   // 0 if not present
             HasHDigest   bool
             HasDDigest   bool
         }
         ```
       - `func ReadRawPDU(r io.Reader, digestHeader, digestData bool) (*RawPDU, error)`:
         Stage 1: `io.ReadFull(r, bhs[:])` -- read exactly 48 bytes. CRITICAL (Pitfall 6): MUST use io.ReadFull, never conn.Read.
         Stage 2: Parse TotalAHSLength from bhs[4] (in 4-byte words -> multiply by 4), DataSegmentLength from bhs[5:8] (24-bit big-endian).
         Stage 3: Compute total remaining bytes:
           `remaining = ahsLen + (if digestHeader: 4 else 0) + dsLen + padLen(dsLen) + (if digestData && dsLen > 0: 4 else 0)`
         Stage 4: `io.ReadFull(r, payload)` for all remaining bytes in ONE call.
         Stage 5: Slice payload into AHS, header digest, data segment, padding (discard), data digest.
         Stage 6: Copy data segment into new caller-owned buffer (D-05). Return pool scratch buffer.

       - `func WriteRawPDU(w io.Writer, p *RawPDU) error`:
         Assemble BHS + AHS + optional header digest + data + padding zeros + optional data digest into a single buffer, then one `w.Write()` call. Single write prevents interleaving (Pitfall 7 -- though the write pump also enforces this).

       - Use `encoding/binary.BigEndian.PutUint32` for writing digest values to wire.

  <behavior>
    - Router.Register allocates a unique ITT and returns a response channel
    - Router.Dispatch delivers a PDU to the channel registered for that ITT
    - Router.Dispatch returns false for unregistered ITT
    - Router.Unregister removes pending ITT entry
    - Router never allocates ITT 0xFFFFFFFF (reserved per RFC 7143)
    - Write pump: goroutine reads from channel, calls WriteRawPDU, serializes all writes
    - Read pump: goroutine calls ReadRawPDU in loop, dispatches via Router by ITT from BHS bytes 16-19
    - Concurrent send from 10 goroutines through write pump produces no interleaving under -race
    - Read pump shutdown when connection closes returns without panic
    - Write pump shutdown when context cancelled drains channel and returns
  </behavior>
  <action>
    1. Create `internal/transport/router.go`:
       ```go
       type Router struct {
           mu      sync.Mutex
           pending map[uint32]chan<- *RawPDU
           nextITT uint32
       }

       func NewRouter() *Router {
           return &Router{pending: make(map[uint32]chan<- *RawPDU)}
       }
       ```
       - `func (r *Router) Register() (uint32, <-chan *RawPDU)` -- allocates next ITT (skipping 0xFFFFFFFF which is reserved for unsolicited target PDUs per RFC 7143), creates buffered channel (cap 1), stores in pending map, returns ITT and receive-only channel.
       - `func (r *Router) Dispatch(itt uint32, pdu *RawPDU) bool` -- locks, looks up ITT in pending, deletes entry, sends PDU on channel, returns true. Returns false if ITT not found (unsolicited PDU or unknown ITT).
       - `func (r *Router) Unregister(itt uint32)` -- removes entry from pending map (for timeout/cancel cleanup).
       - `func (r *Router) PendingCount() int` -- returns len(pending) under lock. For diagnostics.

    2. Create `internal/transport/pump.go` with read and write pump goroutines:
       ```go
       // WritePump owns all writes to the underlying connection.
       // Receives PDUs from writeCh and serializes them to the wire.
       // Returns when ctx is cancelled or writeCh is closed.
       func WritePump(ctx context.Context, w io.Writer, writeCh <-chan *RawPDU) error
       ```
       WritePump implementation:
       - Loop: select on ctx.Done() or writeCh
       - On PDU from writeCh: call WriteRawPDU(w, pdu)
       - On write error: return error (caller handles connection teardown)
       - On ctx.Done(): return ctx.Err()

       ```go
       // ReadPump continuously reads PDUs from the connection and dispatches them.
       // Extracts ITT from BHS bytes 16-19 and calls router.Dispatch.
       // PDUs with ITT=0xFFFFFFFF (unsolicited) are sent to unsolicitedCh.
       // Returns when read fails (connection closed) or ctx is cancelled.
       func ReadPump(ctx context.Context, r io.Reader, router *Router, unsolicitedCh chan<- *RawPDU, digestHeader, digestData bool) error
       ```
       ReadPump implementation:
       - Loop: call ReadRawPDU(r, digestHeader, digestData)
       - Extract ITT from BHS bytes 16-19: `binary.BigEndian.Uint32(pdu.BHS[16:20])`
       - If ITT == 0xFFFFFFFF: send to unsolicitedCh (for NOP-In pings, async messages from target)
       - Otherwise: call router.Dispatch(itt, pdu). If Dispatch returns false, log warning (unexpected ITT).
       - On read error: return error
       - Check ctx between iterations for cancellation

       CRITICAL (Pitfall 7): Only WritePump writes to the connection. All senders put RawPDU on writeCh. This prevents TCP byte interleaving.
