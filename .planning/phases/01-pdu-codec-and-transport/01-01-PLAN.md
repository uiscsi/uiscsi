  </behavior>
  <action>
    1. Create `internal/digest/crc32c.go` in package `digest` with:
       - Package-level `var crc32cTable = crc32.MakeTable(crc32.Castagnoli)` (computed once at init)
       - `func HeaderDigest(bhsAndAHS []byte) uint32` -- returns `crc32.Checksum(bhsAndAHS, crc32cTable)`. Computes CRC32C over BHS + AHS bytes. The 4-byte digest value itself is NOT included in the input. Per RFC 7143 Section 12.1.
       - `func DataDigest(data []byte) uint32` -- computes CRC32C over data segment PLUS zero-padding to 4-byte boundary. Per RFC 7143, data digest covers the data segment and its padding. Implementation:
         ```
         padLen := (4 - (len(data) % 4)) % 4
         if padLen == 0 {
             return crc32.Checksum(data, crc32cTable)
         }
         h := crc32.New(crc32cTable)
         h.Write(data)
         h.Write(make([]byte, padLen))
         return h.Sum32()
         ```
       CRITICAL (Pitfall 5): DataDigest MUST include padding zeros in the CRC computation. Data digest is computed over `data + padding`, not just `data`.
       CRITICAL (Pitfall 4): HeaderDigest covers BHS + ALL AHS bytes. The caller is responsible for passing the correct slice.

       Imports: `hash/crc32` only.
