<objective>
Fix two library-level bugs discovered during UAT:
1. OpReject (0x3F) PDU is unhandled in both the unsolicited PDU dispatcher and the per-task dispatch loop, causing connection drops and reconnect loops.
2. SCSI Response PDU sense data is passed raw (with 2-byte SenseLength prefix per RFC 7143 Section 11.4.7.2) to Result.SenseData, but ParseSense expects raw sense data without the prefix — resulting in all-zero SenseKey/ASC/ASCQ.

Purpose: These are prerequisite library fixes for the E2E test gap closure. Without OpReject handling, negotiation matrix tests loop. Without sense data parsing, SCSI error tests see all-zero fields.
Output: Corrected session dispatch and sense data extraction in the session package.

Fix the sense data extraction in `handleSCSIResponse` (internal/session/datain.go, around line 126).

Per RFC 7143 Section 11.4.7.2, the SCSI Response PDU data segment format when sense data is present:
```
Byte 0-1: SenseLength (big-endian uint16) — number of sense bytes following
Byte 2+:  Actual sense data (SenseLength bytes)
```

Currently the code does:
```go
SenseData: resp.Data,  // raw data segment including SenseLength prefix
```

Change to strip the 2-byte SenseLength prefix:
```go
// Extract sense data from SCSI Response data segment.
// Per RFC 7143 Section 11.4.7.2, the data segment starts with
// a 2-byte SenseLength field followed by the actual sense data.
var senseData []byte
if len(resp.Data) >= 2 {
    senseLen := binary.BigEndian.Uint16(resp.Data[0:2])
    if int(senseLen) <= len(resp.Data)-2 {
        senseData = resp.Data[2 : 2+int(senseLen)]
    }
}
```

Then use `senseData` instead of `resp.Data` in the Result:
```go
r := Result{
    Status:    resp.Status,
    SenseData: senseData,
    // ... rest unchanged
}
```

Add `"encoding/binary"` to the import block if not already present.
