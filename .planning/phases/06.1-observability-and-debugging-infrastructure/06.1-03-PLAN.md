    **Info level (D-13) -- lifecycle transitions:**

    1. In `NewSession`, after creating the session and before starting pumps:
    ```go
    s.cfg.logger.Info("session: opened",
        "isid", fmt.Sprintf("%x", params.ISID),
        "tsih", params.TSIH,

    ```

    2. In `Close`, inside closeOnce.Do after setting wasLoggedIn:
    ```go
    s.cfg.logger.Info("session: closing",
        "was_logged_in", wasLoggedIn)
    ```

    3. In `window.update` or wherever command window changes -- add logging in the `cmdWindow.update` method or after calling it. Check `internal/session/cmdwindow.go` for the right place. The most practical place is in `taskLoop` after `s.window.update(expCmdSN, maxCmdSN)` calls. However, to avoid logging on every PDU, only log when MaxCmdSN actually changes:
    ```go
    oldMax := s.window.maxCmdSN() // need getter
    s.window.update(p.ExpCmdSN, p.MaxCmdSN)
    if s.cfg.logger.Enabled(ctx, slog.LevelInfo) && s.window.maxCmdSN() != oldMax {
        s.cfg.logger.Info("session: command window updated",
            "exp_cmd_sn", p.ExpCmdSN,
            "max_cmd_sn", p.MaxCmdSN)
    }
    ```
    ACTUALLY: Since taskLoop doesn't have ctx, use `context.Background()` or add a sessionCtx field. Simpler: just use the non-context variant `s.cfg.logger.Info(...)` without DebugContext. For Info-level logging the context is less critical.

    4. In `readPumpLoop`, when digest error triggers reconnect -- add Warn per D-14:
    ```go
    import "github.com/rkujawa/uiscsi/internal/digest"

    // In readPumpLoop, check if err is DigestError before reconnect/fatal:
    var de *digest.DigestError
    if errors.As(err, &de) {
        s.cfg.logger.Warn("session: digest mismatch",
            "digest_type", de.Type.String(),
            "expected", fmt.Sprintf("0x%08X", de.Expected),
            "actual", fmt.Sprintf("0x%08X", de.Actual))
    }
    ```

    **Info level -- reconnection (in recovery.go):**

    5. In `triggerReconnect` (or wherever reconnection starts):
    ```go
    s.cfg.logger.Info("session: reconnect started",
        "attempt", attemptNumber,
        "target", s.targetAddr)
    ```

    6. On reconnection success:
    ```go
    s.cfg.logger.Info("session: reconnect complete",
        "attempt", attemptNumber,
        "new_tsih", newTSIH)
    ```

    7. On reconnection failure:
    ```go
    s.cfg.logger.Warn("session: reconnect failed",
        "attempt", attemptNumber,
        "error", err.Error())
    ```

    Read `internal/session/recovery.go` to find exact function names and insertion points.
