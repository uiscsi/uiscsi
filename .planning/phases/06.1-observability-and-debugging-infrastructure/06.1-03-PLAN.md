<objective>
Add structured slog logging throughout session lifecycle and login stages, enrich error messages with protocol context, and ensure DigestError is treated as connection-fatal.

Purpose: Makes protocol issues diagnosable from logs alone (D-12 through D-15). Login stage transitions (D-13) are logged in the login package where they occur. Enriched errors with CmdSN/StatSN/ITT help pinpoint exactly which command and sequence state caused a failure. DigestError is verified as connection-fatal per D-03. This completes the observability layer needed before E2E testing in Phase 7.
Output: Info-level lifecycle and login logging, Warn-level digest/drop logging, enriched error messages, DigestError fatality test.
</objective>

    ```go
    import "github.com/rkujawa/uiscsi/internal/digest"

    // In readPumpLoop, after ReadPump returns error:
    if err != nil && ctx.Err() == nil {
        var de *digest.DigestError
        if errors.As(err, &de) {
            s.cfg.logger.Warn("session: digest mismatch, dropping connection (per D-03)",
                "digest_type", de.Type.String(),
                "expected", fmt.Sprintf("0x%08X", de.Expected),
                "actual", fmt.Sprintf("0x%08X", de.Actual))
            // DigestError is connection-fatal at ERL 0 (D-03).
            // Do NOT attempt reconnect for digest errors -- the connection
            // is corrupt. Set session error directly.
            s.mu.Lock()
            if s.err == nil {
                s.err = fmt.Errorf("session: digest mismatch (connection fatal): %w", err)
            }
            s.mu.Unlock()
            return
        }
        // Non-digest errors follow existing reconnect/fatal logic:
        if s.targetAddr != "" {
            s.triggerReconnect(fmt.Errorf("session: read pump: %w", err))
        } else {
            s.mu.Lock()
            if s.err == nil {
                s.err = fmt.Errorf("session: read pump: %w", err)
            }
            s.mu.Unlock()
        }
    }
    ```
    IMPORTANT: This changes the behavior for DigestError -- it becomes unconditionally fatal (no reconnect attempt), which is correct per RFC 7143 Section 7.3 and D-03.
