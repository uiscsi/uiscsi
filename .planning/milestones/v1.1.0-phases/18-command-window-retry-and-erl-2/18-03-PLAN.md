**Part A: Production code fix in internal/session/recovery.go**

Modify triggerReconnect to dispatch to replaceConnection when params.ErrorRecoveryLevel >= 2:

```go
func (s *Session) triggerReconnect(cause error) {
    s.mu.Lock()
    if s.recovering {
        s.mu.Unlock()
        return
    }
    s.recovering = true
    erl := s.params.ErrorRecoveryLevel
    s.mu.Unlock()
    s.cfg.logger.Info("session: reconnect started",
        "target", s.targetAddr,
        "cause", cause.Error(),
        "erl", erl)
    if erl >= 2 {
        go func() {
            if err := s.replaceConnection(cause); err != nil {
                s.cfg.logger.Error("session: ERL 2 connection replacement failed, falling back to ERL 0", "err", err)
                s.reconnect(cause)
            }
        }()
    } else {
        go s.reconnect(cause)
    }
}
```

This is a minimal change: read s.params.ErrorRecoveryLevel under the existing mutex, dispatch to replaceConnection for ERL >= 2 with fallback to ERL 0 reconnect on failure.

Verify existing tests still pass: `go test ./internal/session/ -race -count=1 -timeout 60s`

**TestERL2_ConnectionReassignment (SESS-07, per D-01 and D-02):**
This tests FFP #7.1: after a TCP connection drop with ERL=2, the initiator dials a new connection to the same target with the same ISID+TSIH, performs login, and sends Logout(reasonCode=2).

CRITICAL: The discriminating signal between ERL 2 and ERL 0 is Logout(reasonCode=2)
on the wire. ERL 0 reconnect does NOT send Logout(reasonCode=2). The test MUST assert
this specific PDU is captured via pducapture.

callCount scope: The atomic counter in HandleSCSIFunc is per-MockTarget (closure-scoped),
NOT per-connection. So callCount==0 fires on the first connection, and callCount==1
fires on the second connection's first SCSI command. This is correct for this test.

- Create MockTarget with HandleLogin, HandleLogout, HandleNOPOut.
- SetNegotiationConfig with ErrorRecoveryLevel=2.
- Attach pducapture.Recorder.
- Register a TMF handler (pdu.OpTaskMgmtReq) that responds with TMF Response (Function Complete, response=0x00) for TASK REASSIGN (function=14). For other TMF functions, respond with Function Complete as well.
- HandleSCSIFunc:
  - callCount==0: Do NOT send a response. Instead, close the TCP connection (tc.Close() or close the underlying net.Conn) to simulate a connection drop while a command is in-flight. This triggers the initiator's read pump error, which (after the production fix) dispatches to replaceConnection.
  - callCount>=1: Respond normally with Data-In (HasStatus=true, Status=0x00, 512 bytes).
- Dial with WithPDUHook(rec.Hook()), WithKeepaliveInterval(30s), WithOperationalOverrides({"ErrorRecoveryLevel": "2"}).
- Issue ReadBlocks in a goroutine (it will block while connection drops and recovery happens).
- Wait for the ReadBlocks to complete (up to 15s timeout). It should succeed after ERL 2 recovery re-issues the command.
- Verify via pducapture (THIS IS THE KEY DISCRIMINATING ASSERTION):
  - Sent(pdu.OpLogoutReq) must contain at least 1 Logout PDU.
  - Type-assert the captured Logout PDU to *pdu.LogoutReq.
  - Assert ReasonCode == 2 (remove connection for recovery). This is the ERL 2 discriminating signal -- ERL 0 reconnect does NOT send Logout(reasonCode=2).
  - If no Logout(reasonCode=2) found: t.Fatal("SESS-07: no Logout(reasonCode=2) on wire -- ERL 2 path was not taken; likely fell back to ERL 0")
  - At least 2 Login Request PDUs were sent (initial + reconnect).
- If ReadBlocks fails, log the error and check if the recovery path was attempted.

**TestERL2_TaskReassign (SESS-08, per D-01):**
This tests FFP #19.5: after ERL 2 connection replacement, the initiator sends TMF TASK REASSIGN (Function=14) for each in-flight task, with ReferencedTaskTag set to the original ITT.

callCount scope: Per-MockTarget (closure-scoped atomic counter). callCount==0 fires on
first connection (drop without responding), callCount==1 fires on second connection's
first SCSI command (respond normally). No risk of infinite drop loop.

- This test may share much of the setup from TestERL2_ConnectionReassignment. Refactor common setup into a helper if beneficial, or duplicate setup for clarity.
- Create MockTarget with HandleLogin, HandleLogout, HandleNOPOut.
- SetNegotiationConfig with ErrorRecoveryLevel=2.
- Attach pducapture.Recorder.
- Register TMF handler (pdu.OpTaskMgmtReq) that:
  - Checks if Function==14 (TASK REASSIGN).
  - If so, record the ReferencedTaskTag, then respond with TMF Response (response=0x00 Function Complete).
  - For other functions, respond normally.
- HandleSCSIFunc:
  - callCount==0: Record the InitiatorTaskTag (this is the original ITT of the in-flight command). Close the connection without responding.
  - callCount>=1: Respond normally with Data-In (HasStatus=true, Status=0x00, 512 bytes).
- Dial with WithPDUHook(rec.Hook()), WithKeepaliveInterval(30s), WithOperationalOverrides({"ErrorRecoveryLevel": "2"}).
- Issue ReadBlocks in a goroutine.
- Wait for completion (up to 15s).
- Verify via pducapture:
  - Sent(pdu.OpTaskMgmtReq) contains at least 1 TMF PDU.
  - Assert at least one TMF PDU has Function=14 (TASK REASSIGN). To check this, type-assert the captured PDU to *pdu.TaskMgmtReq and verify tmf.Function == 14.
  - Verify ReferencedTaskTag in the TMF matches the original ITT from the first SCSI Command (Sent(pdu.OpSCSICommand)[0].InitiatorTaskTag).
  - Verify the ReadBlocks eventually succeeds (the task was reassigned and completed on the new connection).
