**TestRetry_ExpStatSNGap (CMDSEQ-08, per D-06):**

This tests FFP #5.1: when target sends a SCSI Response with a StatSN jumped by N, the
initiator detects the gap. CRITICAL: updateStatSN() in session.go simply advances
expStatSN with no gap detection. The SNACK timer fires on "tail loss" (no response
arrives at all), NOT on StatSN gaps in received responses.

Strategy: Issue two sequential read commands. For the SECOND command, do NOT respond
(close the connection or simply never send a response). This creates a tail loss
scenario where the SNACK timer should fire a Status SNACK at ERL>=1. We also jump
the StatSN in the first response to verify the initiator handles both conditions.

IMPORTANT: The test MUST assert that a Status SNACK PDU (Type=1) is sent on the wire.
If no Status SNACK is captured within 5 seconds, the test MUST fail explicitly with:
  t.Fatal("CMDSEQ-08: no Status SNACK sent after StatSN gap -- production code does not implement StatSN gap detection via SNACK at ERL=1")
Do NOT accept "graceful gap tolerance" (session surviving without SNACK) as a pass.

Implementation:
- Create MockTarget with HandleLogin, HandleLogout, HandleNOPOut.
- SetNegotiationConfig with ErrorRecoveryLevel=1.
- Attach pducapture.Recorder.
- Register SNACK handler (pdu.OpSNACKReq) that:
  - Type-asserts to *pdu.SNACKReq
  - If Type==1 (Status SNACK): responds with a normal SCSI Response for the missing status (or simply logs it). Record that a Status SNACK was received.
  - For other types: silently consume.
- HandleSCSIFunc:
  - callCount==0: Call tgt.Session().Update(). Advance StatSN by calling tc.NextStatSN() 5 extra times to skip 5 values (creating a StatSN gap of 5). Send SCSI Response with the jumped StatSN, HasStatus=true, Status=0x00, 512 bytes data. This creates a gap in the initiator's ExpStatSN tracking.
  - callCount==1: Do NOT respond. The command hangs, which triggers the SNACK timer for tail loss.
  - callCount>=2: Respond normally.
- Dial with WithPDUHook(rec.Hook()), WithKeepaliveInterval(30s), WithOperationalOverrides({"ErrorRecoveryLevel": "1"}).
- First ReadBlocks: should succeed (response has good status, just jumped StatSN).
- Second ReadBlocks in a goroutine with 10s context timeout. This will hang (no response from target), triggering the SNACK timer.
- Wait up to 5s, then check pducapture for Status SNACK PDUs: Sent(pdu.OpSNACKReq).
- Assert at least one SNACK PDU with Type==1 (Status SNACK) was sent.
- If no Status SNACK found: t.Fatal("CMDSEQ-08: no Status SNACK sent -- ERL=1 status recovery via SNACK timer is not functioning for tail loss scenario")
- Cancel the second ReadBlocks context to clean up.

Note on tc.NextStatSN(): Call it in a loop to skip values. If TargetConn does not
expose direct StatSN manipulation, call tc.NextStatSN() 5 times and discard results,
then use the 6th call for the actual response StatSN. Read TargetConn to confirm.
