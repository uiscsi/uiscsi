AsyncEvent codes per RFC 7143 Section 11.9:
- Code 1: Target requests logout. Parameter3 = seconds to logout.
- Code 2: Connection drop. Parameter1 = CID, Parameter2 = Time2Wait, Parameter3 = Time2Retain.
- Code 3: Session drop. Parameter2 = Time2Wait, Parameter3 = Time2Retain.
- Code 4: Negotiation request. Parameter3 = seconds to send TextReq.

1. LogoutReq appears on wire within 1 second (capture timestamp or just verify it appears within 2s sleep)

3. Use `WithAsyncHandler` to capture the async event callback and verify EventCode=1

This differentiates from SESS-01 by testing the behavioral constraint (no new commands) and timing, not the LogoutReq wire fields.

**TestAsync_ConnectionDrop (ASYNC-02, FFP #20.2):**
Tests that AsyncEvent code 2 triggers reconnect or error.

Setup: HandleLogin, HandleLogout (both needed for clean reconnect), HandleSCSIFunc. On first SCSI command, respond normally then inject `SendAsyncMsg(tc, 2, AsyncParams{Parameter1: 0, Parameter2: 0, Parameter3: 0})` (CID=0, immediate reconnect).

1. The async handler callback receives EventCode=2 via WithAsyncHandler

Recommended: Verify the error path. After async injection and brief sleep, call `sess.TestUnitReady(ctx, 0)` and assert it returns an error. The error should indicate connection/session failure.
