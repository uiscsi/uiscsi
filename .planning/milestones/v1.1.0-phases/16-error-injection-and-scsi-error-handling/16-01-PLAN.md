**ERR-05: TestError_BUSY** (simplest, do first)
- Use `tgt.HandleSCSIWithStatus(0x08, nil)` -- BUSY status, no sense data.
- Dial, issue sess.ReadBlocks, verify errors.As yields `*SCSIError` with Status==0x08 (per D-04).
- No sense data expected -- SenseKey/ASC/ASCQ will be zero.

**ERR-06: TestError_ReservationConflict**
- Use `tgt.HandleSCSIWithStatus(0x18, nil)` -- RESERVATION CONFLICT.
- Same pattern as ERR-05 but verify Status==0x18 (per D-04).

**ERR-03: TestError_UnexpectedUnsolicited**
- Build sense data: fixed format (0x70), sense key=0x0B (ABORTED COMMAND), ASC=0x0C (WRITE ERROR), ASCQ=0x0D (UNEXPECTED UNSOLICITED DATA).
- Use `tgt.HandleSCSIWithStatus(0x02, senseData)` -- CHECK CONDITION with sense.
- Dial, issue sess.WriteBlocks (write path triggers unsolicited data), verify errors.As yields `*SCSIError` with Status==0x02, SenseKey==0x0B, ASC==0x0C, ASCQ==0x0D.

**ERR-04: TestError_NotEnoughUnsolicited**
- Same as ERR-03 but ASC=0x0C, ASCQ=0x0E (NOT ENOUGH UNSOLICITED DATA).
- Verify Status==0x02, SenseKey==0x0B, ASC==0x0C, ASCQ==0x0E.

**ERR-01: TestError_CRCErrorSense**
- Build sense data: fixed format (0x70), sense key=0x0B (ABORTED COMMAND), ASC=0x47 (SCSI PARITY ERROR), ASCQ=0x05 (CRC ERROR DETECTED).
- Use HandleSCSIFunc (per D-01 -- CRC error is a complex scenario that tests sense parsing).
- Handler builds SCSIResponse with CHECK CONDITION (0x02) and the CRC sense data, using SessionState.Update for CmdSN.
- Verify errors.As yields `*SCSIError` with Status==0x02, SenseKey==0x0B, ASC==0x47, ASCQ==0x05.

**ERR-02: TestError_SNACKRejectNewCommand**
- This is the most complex test. Uses HandleSCSIFunc with callCount per D-01.
- Configure ERL=1 on both target (SetNegotiationConfig) and initiator (WithOperationalOverrides).
- Setup PDU recorder with WithPDUHook.
- callCount==0 (first SCSI command):
  1. Send Data-In with DataSN gap: send DataSN=0 (512 bytes), skip DataSN=1, send DataSN=2 (512 bytes). This triggers initiator SNACK.
  2. Use tc.ReadPDU() to receive the SNACK PDU from the initiator.
  3. Build Reject PDU: Reason=0x09 (Invalid PDU Field), ITT=0xFFFFFFFF, StatSN=tc.NextStatSN(), Data=SNACK's BHS bytes.
  4. Send Reject via tc.SendPDU.
  5. The first command should fail (Reject cancels the task).
- callCount==1 (second SCSI command -- retry by caller):
  1. Send normal Data-In with status=0x00 for 512 bytes.
  2. This should succeed.
- Test code: first sess.ReadBlocks should return error; second sess.ReadBlocks should succeed.
- Verify the first error is not nil (Reject causes task cancellation).
- Important: The target needs HandleLogin, HandleLogout, HandleNOPOut registered.
- Use `time.Sleep(100*time.Millisecond)` after first ReadBlocks to let async processing settle.

All tests use 10-second context timeout. All tests use t.Cleanup for sess.Close and tgt.Close.
