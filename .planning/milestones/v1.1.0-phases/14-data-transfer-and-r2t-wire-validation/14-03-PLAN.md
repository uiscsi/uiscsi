**Test 1: TestDataIn_StatusInFinal** (DATA-06, FFP #10.1)
- HandleSCSIFunc: send 3 Data-In PDUs (512 bytes each for 1536 total). First two: Final=false, HasStatus=false. Last: Final=true, HasStatus=true, Status=0x00, StatSN incremented.
- Read 1536 bytes via sess.Read().
- Assert from rec.Received(pdu.OpDataIn): last PDU has Final=true AND HasStatus=true AND Status==0. Earlier PDUs have Final=false AND HasStatus=false.
- Assert: Read() returns 1536 bytes without error (status accepted inline).

**Test 2: TestDataIn_ABitDataACK** (DATA-07, FFP #10.2)
- Config: ErrorRecoveryLevel=1 on both sides (target NegotiationConfig + initiator WithOperationalOverrides).
- HandleSCSIFunc: send 3 Data-In PDUs. Set Acknowledge=true (A-bit) on the 2nd PDU. Last PDU has S+F.
- Read data via sess.Read().
- Assert: rec.Sent(pdu.OpSNACKReq) has at least 1 SNACK. Decode it: Type field (byte 1 bits 3-1) should be 2 (DataACK). BegRun should be the DataSN following the A-bit PDU.
- Note: The SNACK type is encoded in BHS byte 1 bits 3-1 per RFC 7143 Section 11.16.1. The pdu.SNACKReq struct may encode this differently -- check the PDU struct for the Type field location.

**Test 3: TestDataIn_ZeroLength** (DATA-09, FFP #11.1.2)
- HandleSCSIFunc: send 2 Data-In PDUs. First: 512 bytes, DataSN=0. Second: DataSegmentLen=0, Data=nil, DataSN=1, Final=true, HasStatus=true.
- Read 512 bytes via sess.Read().
- Assert: Read() returns 512 bytes without error. No panic or error from zero-length Data-In.

**Test 4: TestDataIn_EDTL** (DATA-14, FFP #16.6)
- HandleSCSIFunc: send 4 Data-In PDUs of 256 bytes each (1024 total). Last has S+F.
- Read 1024 bytes (EDTL=1024).
- Assert: sum of all Data-In DataSegmentLen values from rec.Received(pdu.OpDataIn) == 1024 (matches EDTL). Read() returns exactly 1024 bytes.
