Add 4 more test functions to test/conformance/dataout_test.go for unsolicited Data-Out paths.

**CRITICAL pitfall (Pitfall 5):** Immediate data is in the SCSI Command PDU DataSegment, not a separate Data-Out PDU. When verifying FirstBurstLength compliance, sum: (1) SCSI Command DataSegmentLen (immediate data) from rec.Sent(pdu.OpSCSICommand), plus (2) all unsolicited Data-Out DataSegmentLen values from rec.Sent(pdu.OpDataOut) that appear BEFORE any R2T is received.

**Test 7: TestDataOut_UnsolicitedFirstBurst** (DATA-02, FFP #8.1)
- Config: ImmediateData=Yes, InitialR2T=No on both sides. FirstBurstLength=1024, MaxRecvDSL=512 on target.
- HandleSCSIFunc: after receiving SCSI Command (which carries immediate data), read unsolicited Data-Out PDUs until F-bit. Then send R2T for remaining data. ReadDataOutPDUs for solicited burst. Send SCSIResponse.
- Write 2048 bytes. FirstBurstLength=1024 means up to 1024 bytes unsolicited (512 immediate + 512 unsolicited Data-Out). Remaining 1024 via R2T.
- Assert: sum of SCSI Command DataSegmentLen + unsolicited Data-Out DataSegmentLen <= 1024 (FirstBurstLength). Solicited Data-Out PDUs also present.

**Test 8: TestDataOut_NoUnsolicited** (DATA-03, FFP #8.2)
- Config: ImmediateData=No, InitialR2T=Yes on both sides.
- HandleSCSIFunc: send R2T for full EDTL, ReadDataOutPDUs, send response.
- Write 1024 bytes.
- Assert: SCSI Command DataSegmentLen == 0 (no immediate data). No Data-Out PDUs appear before R2T (all Data-Out has TTT matching R2T's TTT, not 0xFFFFFFFF).

**Test 9: TestDataOut_FirstBurstLimit** (DATA-04, FFP #8.3)
- Config: ImmediateData=Yes, InitialR2T=No. FirstBurstLength=768, MaxRecvDSL=256 on target.
- HandleSCSIFunc: read unsolicited Data-Out until F-bit after SCSI Command. Then send R2T for remaining. ReadDataOutPDUs. Send response.
- Write 2048 bytes. Unsolicited burst: 256 immediate + up to 512 Data-Out = 768 total.
- Assert: sum of immediate data + unsolicited Data-Out DataSegmentLen == 768 (exactly FirstBurstLength, or less if EDTL < FBL). Unsolicited Data-Out PDUs have TTT = 0xFFFFFFFF.

**Test 10: TestDataOut_FBitUnsolicited** (DATA-10, FFP #11.2.1)
- Config: ImmediateData=Yes, InitialR2T=No. FirstBurstLength=1024, MaxRecvDSL=256 on target.
- HandleSCSIFunc: read unsolicited Data-Out until F-bit. Send R2T for remaining. ReadDataOutPDUs. Send response.
- Write 2048 bytes. Unsolicited: 256 immediate + 3x256 Data-Out = 1024.
- Assert: unsolicited Data-Out PDUs (TTT=0xFFFFFFFF): first two have Final=false, last has Final=true.

For unsolicited Data-Out identification: Data-Out PDUs with TargetTransferTag == 0xFFFFFFFF are unsolicited per RFC 7143 Section 11.7.1.
