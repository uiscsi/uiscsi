**Test: TestCmdSN_ImmediateDelivery_NonTMF** (CMDSEQ-02, FFP #2.1)

Per D-08 (field-strict: check Immediate flag and CmdSN), D-09 (t.Error):

The primary non-TMF immediate command is NOP-Out. Per research open question #1, the reliable way to trigger NOP-Out is to have MockTarget send a solicited NOP-In (with TTT != 0xFFFFFFFF) forcing the initiator to respond with NOP-Out.

1. Create Recorder, MockTarget
2. Register HandleLogin, HandleLogout, HandleSCSIFunc (simple TEST UNIT READY handler using SessionState)
3. Replace the default HandleNOPOut with a custom handler: register `tgt.Handle(pdu.OpNOPOut, ...)` that handles the NOP-Out response AND also, on the first SCSI command, sends an unsolicited NOP-In from the target (to trigger initiator NOP-Out response). Alternatively, simpler approach: register HandleNOPOut normally, and rely on the keepalive timer. Set keepalive interval short: `uiscsi.WithKeepaliveInterval(100*time.Millisecond)`.
4. Better approach per research: After HandleSCSIFunc responds to first SCSI command, have the MockTarget send a NOP-In with TTT != 0xFFFFFFFF to the initiator. The initiator MUST respond with NOP-Out. Implementation:
   - In HandleSCSIFunc, after the first SCSI response (callCount==0), use `tc.SendPDU(&pdu.NOPIn{Header: pdu.Header{Final: true}, TargetTransferTag: 0x00000001, StatSN: tc.NextStatSN(), ExpCmdSN: expCmdSN, MaxCmdSN: maxCmdSN})` to solicit a NOP-Out response.
   - Register HandleNOPOut to handle the response (it already sends NOP-In with TTT=0xFFFFFFFF back).
   
   Wait -- HandleNOPOut already handles incoming NOP-Out. The issue is we need the target to INITIATE a NOP-In to force a NOP-Out from the initiator. Do this:
   
   - In HandleSCSIFunc, on callCount==0, after sending SCSI response, send a NOP-In with TTT=0x00000001 (a solicited ping from target). The initiator will respond with NOP-Out echoing TTT.
   - HandleNOPOut is already registered and will handle the NOP-Out that comes back.
   
5. Send `sess.TestUnitReady(ctx, 0)` (triggers SCSI command + NOP-In from target + NOP-Out from initiator)
6. Sleep briefly (50ms) to allow NOP-Out exchange to complete
7. Send `sess.TestUnitReady(ctx, 0)` again (second SCSI command)
8. `sess.Close()`
9. Get `nops := rec.Sent(pdu.OpNOPOut)` -- verify at least 1 NOP-Out was captured
10. For each NOP-Out: type-assert `*pdu.NOPOut`, verify `p.Header.Immediate == true` (t.Errorf per D-09 if false: "NOP-Out should have Immediate=true")
11. Get `cmds := rec.Sent(pdu.OpSCSICommand)` -- verify at least 2 SCSI commands
12. Verify SCSI commands have sequential CmdSN with delta=1 (NOP-Out did NOT consume a CmdSN slot). The two SCSI commands should have CmdSN N and N+1 despite the NOP-Out between them.
