**Add SessionState struct** (after the existing TargetConn type, before MockTarget):

```go
// SessionState tracks target-side command sequencing state.
// Handlers call Update() to get correct ExpCmdSN/MaxCmdSN for responses
// instead of hardcoding cmd.CmdSN+1/cmd.CmdSN+10.
type SessionState struct {
    mu            sync.Mutex
    expCmdSN      uint32
    maxCmdSNDelta int32 // MaxCmdSN = ExpCmdSN + delta; default 10
    initialized   bool
}
```

Methods on SessionState:
- `func NewSessionState() *SessionState` -- returns `&SessionState{maxCmdSNDelta: 10}`
- `func (ss *SessionState) Update(cmdSN uint32, immediate bool) (expCmdSN, maxCmdSN uint32)`:
  - Lock mutex
  - If not initialized: set `ss.expCmdSN = cmdSN`, set `ss.initialized = true`
  - If NOT immediate: `ss.expCmdSN = cmdSN + 1`
  - Compute `maxCmdSN = uint32(int32(ss.expCmdSN) + ss.maxCmdSNDelta)`
  - Return expCmdSN, maxCmdSN
  - Unlock mutex
- `func (ss *SessionState) SetMaxCmdSNDelta(delta int32)`:
  - Lock, set delta, unlock
- `func (ss *SessionState) ExpCmdSN() uint32` -- read-only accessor (lock, read, unlock)

**Add HandleSCSIFunc method to MockTarget** per D-04:

```go
// HandleSCSIFunc registers a flexible SCSI command handler where the test
// function receives the decoded *pdu.SCSICommand and a 0-based call counter.
// This replaces any previously registered OpSCSICommand handler (HandleSCSIRead,
// HandleSCSIWrite, HandleSCSIError).
func (mt *MockTarget) HandleSCSIFunc(h func(tc *TargetConn, cmd *pdu.SCSICommand, callCount int) error) {
    var count atomic.Int32
    mt.Handle(pdu.OpSCSICommand, func(tc *TargetConn, raw *transport.RawPDU, decoded pdu.PDU) error {
        cmd := decoded.(*pdu.SCSICommand)
        n := int(count.Add(1) - 1)
        return h(tc, cmd, n)
    })
}
```

Use `atomic.Int32` for the call counter (not a bare int with mutex) since handler may be called from multiple goroutines if multiple connections exist.

**Update HandleLogin** to initialize SessionState: In the HandleLogin method, after setting the first `ExpCmdSN` and `MaxCmdSN` in login responses, call `mt.session.Update(req.CmdSN, false)` so SessionState is seeded with the initial CmdSN from login. Specifically, add `mt.session.Update(req.CmdSN, false)` at the start of the login handler (before any response is sent), so that `ss.expCmdSN` equals `req.CmdSN + 1` after login completes.

Do NOT migrate existing HandleSCSIRead/HandleSCSIWrite/HandleSCSIError handlers to use SessionState yet (per research recommendation -- they still work for existing tests). Only HandleSCSIFunc and HandleLogin use SessionState.
