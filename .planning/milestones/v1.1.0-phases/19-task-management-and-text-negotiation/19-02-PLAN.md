**Test 1: TestText_Fields (TEXT-01)**
- Per D-04 pattern: trigger renegotiation via AsyncMsg code 4
- Setup: HandleText() (default echo handler), HandleLogin, HandleNOPOut
- Send TestUnitReady to trigger SCSI handler which injects AsyncMsg
- Poll for TextReq in pducapture
- Assert opcode is OpTextReq (0x04)
- Assert F-bit (Final) is true (single exchange, not continuation)
- Assert data segment is not empty and contains valid KV format: key=value pairs separated by null bytes
- Assert data segment contains at least one of: MaxRecvDataSegmentLength, MaxBurstLength, FirstBurstLength (these are what renegotiate() proposes)

**Test 2: TestText_ITTUniqueness (TEXT-02)**
- Per D-04: Sequential renegotiation calls for ITT uniqueness
- Setup: HandleText (echo), HandleLogin, HandleNOPOut
- Trigger 3 renegotiations via AsyncMsg code 4 (HandleSCSIFunc injects on callCount 0, 1, 2)
- Send 3 TestUnitReady calls to trigger 3 async injections
- Wait for 3 TextReqs in pducapture (poll until count >= 3)
- Collect all ITTs into a set
- Assert all 3 ITTs are distinct (no duplicates)
- Assert none equals 0xFFFFFFFF (reserved)

**Test 3: TestText_TTTInitial (TEXT-03)**
- Trigger renegotiation via AsyncMsg code 4
- Capture first TextReq
- Assert TTT (TargetTransferTag) == 0xFFFFFFFF
- This verifies the initiator uses 0xFFFFFFFF for initiator-initiated exchanges

**Test 4: TestText_TTTContinuation (TEXT-04)**
- Per D-05: Custom HandleText returns partial response with C=1 and TTT=0x12345678
- IMPORTANT: Use SendTargets (not AsyncMsg code 4) because renegotiate() does NOT handle continuation (Pitfall 1 from RESEARCH.md)
- Register custom Handle(pdu.OpTextReq, ...) that:
  - On first request (TTT==0xFFFFFFFF): responds with Continue=true, TTT=0x12345678, partial data ("SendTargets=\0" or similar), Final=false
  - On continuation (TTT==0x12345678): responds with Continue=false, TTT=0xFFFFFFFF, final data with a target entry, Final=true
- Call sess.SendTargets(ctx) -- this handles continuation loop
- Capture all TextReqs via pducapture
- Assert first TextReq has TTT=0xFFFFFFFF (initial)
- Assert second TextReq has TTT=0x12345678 (echoes target's TTT)
- Assert second TextReq F-bit is true (final in continuation sequence)

**Test 5: TestText_OtherParams (TEXT-05)**
- Trigger renegotiation via AsyncMsg code 4
- Capture TextReq
- Assert Immediate bit is false (TextReq is NOT immediate -- it acquires CmdSN)
- Assert CmdSN is a valid value (> 0 or matches expected sequence)
- Assert ExpStatSN is valid (matches session's expected StatSN)
- Verify CmdSN is acquired (non-immediate) by checking that the SCSI command before has CmdSN = TextReq.CmdSN - 1 (or offset by number of commands sent)

**Test 6: TestText_NegotiationReset (TEXT-06)**
- Per D-06: Fresh ITT + TTT after completed exchange
- Trigger two complete renegotiations (AsyncMsg code 4 twice)
- After first exchange completes (TTT=0xFFFFFFFF in response), trigger second
- Capture both TextReqs
- Assert second TextReq has different ITT from first (fresh ITT)
- Assert second TextReq has TTT=0xFFFFFFFF (no stale state, fresh exchange)
- This proves negotiation state is properly reset between exchanges

All tests use `pducapture.Recorder` with `uiscsi.WithPDUHook(rec.Hook)`. All use the async injection polling pattern from async_test.go (100ms poll, 5s deadline). Use `testutil.NewMockTarget()` + `tgt.HandleLogin()` + `tgt.HandleNOPOut()`.

For TEXT-01/02/03/05/06, use AsyncMsg code 4 to trigger TextReq. For TEXT-04, use SendTargets() with custom HandleText for continuation.
