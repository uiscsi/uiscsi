**Step 1: Fix sendTMF for RefCmdSN (if needed per TMF-03/D-03)**

Read `internal/session/tmf.go`. The `sendTMF` function currently does NOT set RefCmdSN in the TaskMgmtReq PDU. Per RFC 7143 Section 11.5.1, AbortTask must carry the CmdSN of the referenced task in the RefCmdSN field. Modify `sendTMF` to accept an optional `refCmdSN` parameter (or add a new `sendTMFWithRefCmdSN` variant), and have `AbortTask` pass the referenced task's CmdSN.

The challenge: `AbortTask` currently only receives `targetITT` (the ReferencedTaskTag). It does not know the CmdSN of the referenced task. The initiator would need to track CmdSN per task (it already tracks ITT via Router). Look at how tasks are registered in Router -- if CmdSN is not stored per task, RefCmdSN=0 may be the current state. In that case, document this as a known limitation and have the test verify the actual behavior (RefCmdSN=0 for AbortTask), noting that RFC 7143 says RefCmdSN is only relevant for AbortTask and the field is reserved for other TMFs.

**IMPORTANT:** If the production code does not track per-task CmdSN and cannot easily populate RefCmdSN, do NOT add complex tracking infrastructure. Instead: (1) verify TMF-03 by checking that RefCmdSN field EXISTS in the PDU and is a valid uint32, and (2) document the limitation. The test should verify the wire format is correct even if the value is 0.

**Step 2: Create test/conformance/tmf_test.go with all 6 TMF tests**

Package: `conformance_test`. Imports: same as erl2_test.go pattern.

**Test 1: TestTMF_CmdSN (TMF-01)**
- Per D-01: Stall HandleSCSIFunc on callCount==0 to create in-flight task
- Send a SCSI command first (TestUnitReady) to establish CmdSN baseline
- Send AbortTask targeting the stalled task
- Capture TaskMgmtReq via `rec.Sent(pdu.OpTaskMgmtReq)`
- Assert: TMF CmdSN equals the NEXT expected CmdSN (same as current window value since TMF is immediate and does NOT acquire)
- Assert: TMF Immediate bit is true
- Unblock stalled handler, verify no errors

**Test 2: TestTMF_LUNEncoding (TMF-02)**
- Per D-02 (scoped by research Pitfall 6): Test flat space encoding for LUN values 0, 1, and a higher value (e.g., 3)
- For each LUN: send LUNReset (simplest LUN-scoped TMF, no stall needed)
- Capture TaskMgmtReq, assert LUN field matches `pdu.EncodeSAMLUN(targetLUN)`
- Use table-driven subtest: `t.Run(fmt.Sprintf("LUN_%d", lun), ...)`
- HandleTMF() for standard response

**Test 3: TestTMF_RefCmdSN (TMF-03)**
- Per D-03: Only AbortTask carries RefCmdSN
- Per D-01: Stall HandleSCSIFunc on callCount==0
- Launch ReadBlocks in goroutine (blocks in stalled handler)
- Capture the SCSI command's CmdSN from pducapture: `rec.Sent(pdu.OpSCSICommand)`
- Send AbortTask targeting captured ITT
- Capture TaskMgmtReq, verify RefCmdSN field
- Verify ReferencedTaskTag == captured SCSI ITT
- Verify TMF's own ITT != ReferencedTaskTag (per research Pitfall 3)

**Test 4: TestTMF_AbortTaskSet_AllTasks (TMF-04)**
- Stall HandleSCSIFunc on callCount 0 AND 1 (two in-flight tasks on same LUN)
- Launch 2 ReadBlocks in goroutines
- Wait for both to be in-flight (time.Sleep 200ms)
- Custom HandleTMF that sends Function Complete AND unblocks stalled handlers
- Call sess.AbortTaskSet(ctx, lun)
- Verify both ReadBlocks goroutines return with error (ErrTaskAborted or similar)
- Capture TaskMgmtReq, verify Function==2 (AbortTaskSet)

**Test 5: TestTMF_AbortTaskSet_BlocksNew (TMF-05)**
- Per D-07: Goroutine + timer blocking proof
- Stall HandleSCSIFunc on callCount==0 (one in-flight task)
- Launch ReadBlocks in goroutine (creates in-flight task)
- Wait for it to be in-flight (200ms)
- Custom HandleTMF that DELAYS response (channel wait)
- Call sess.AbortTaskSet in goroutine
- Wait 100ms, then launch NEW ReadBlocks in separate goroutine
- Verify new ReadBlocks does NOT complete within 300ms (blocked)
- Unblock TMF handler (send TMF Response), also unblock stalled SCSI handler
- Verify new ReadBlocks eventually completes (within 5s)
- NOTE: If the initiator does NOT block new commands during AbortTaskSet, this test will FAIL. In that case, the test reveals a production code gap. Document in SUMMARY that TMF-05 blocking is not implemented and needs follow-up.

**Test 6: TestTMF_AbortTaskSet_ResponseAfterClear (TMF-06)**
- Per D-08: Immediate TMF Response
- Stall HandleSCSIFunc on callCount==0 (one in-flight task)
- Launch ReadBlocks in goroutine (creates in-flight task)
- Wait for it to be in-flight (200ms)
- Custom HandleTMF sends immediate Function Complete + unblocks stalled handler
- Call sess.AbortTaskSet
- After AbortTaskSet returns, verify the in-flight ReadBlocks goroutine has completed with an error
- This tests that by the time AbortTaskSet response is processed, in-flight tasks are canceled

All tests use `pducapture.Recorder` with `uiscsi.WithPDUHook(rec.Hook)`. All use `testutil.NewMockTarget()` + `tgt.HandleLogin()` + `tgt.HandleNOPOut()`. Use 300ms for blocking proof timers per Claude's discretion.
